    üõ†Ô∏è How to Write a Good Prompt (G.C.S.E. Framework) ‚ñº

    The G.C.S.E. prompt builder helps you get the best from Copilot:

    • Goal: What do you want Copilot to do?
      Example: "Summarise this FCA policy document for a non-technical audience."
    • Context: Why do you need this? Who is it for?
      Example: "This summary will be shared with new joiners in the Financial Crime team."
    • Source: Where should Copilot look?
      Example: "Use only the attached document and FCA website."
    • Expectations: How should the output look?
      Example: "Present as bullet points, using clear, fair, and not misleading language."
    Full Example:
    "Summarise the attached FCA guidance for new joiners. Use only the document provided. Present the summary as bullet points in plain English, ensuring all points are clear, fair, and not misleading."

    Before using Copilot, check each item (click to mark as complete):

    Am I working on live operations or enforcement matters? (If YES, do NOT use Copilot - CPIA 1996 disclosure risks)
    Is this task relevant to my role and responsibilities?
    Am I using only public or appropriately cleared information?
    Have I verified there's no personal or confidential data?
    Is using Copilot proportionate to this task?
    Will I verify and review all Copilot outputs?
    Have I used language that is clear, fair, and not misleading?
    Does this output align with FCA principles?
